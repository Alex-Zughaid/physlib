# Contributing

Physlib is open-source and community run — contributions from anyone are
welcome. Open a pull request and a maintainer will review it and iterate with
you until it can be merged.

Not sure where to start? Try:

- [Open issues](https://github.com/leanprover-community/physlib/issues)
- [TODO list](https://physlib.io/TODOList)
- [Get Involved page](https://physlib.io/GetInvolved.html)
- The [QuantumInfo TODO page](https://github.com/leanprover-community/physlib/blob/master/QuantumInfo/WildeTODO.md)

If you're stuck, ask on the [Physlib Zulip](https://leanprover.zulipchat.com/#narrow/channel/479953-Physlib).

## Making a pull request

1. Fork the repository.
2. Make your changes in small, reviewable steps — small PRs get reviewed
   faster and it's easier to give useful feedback.
3. Open the PR. It will go through:
   - a review against the [Review-Guidelines](./Review-Guidelines.md)
   - automated linter checks (see the
     [linter guide](https://github.com/leanprover-community/physlib/blob/master/scripts/README.md)
     for running them locally)

Your PR doesn't need to be perfect on arrival — expect roughly half the work
to happen after you open it, through review feedback. That's normal.
